## Overview

Tags are powerful organizational tools that help categorize and quickly identify the type of information in resources. The KnowledgeBank uses a three-tier tag system to ensure quality and consistency in content organization.

![Tags page](/images/guides/tags-page.png)
*The tags page showing all available tags with their usage counts*

## Understanding Tag Types

The system uses four types of tags, each with different levels of authority and importance:

1. **Normal Tags**
    - **Created by:** Any user
    - **Purpose:** Basic categorization and content tagging
    - **Appearance:** Standard text without special indicators
    - **Examples:** "Citizen Council", "Citizen Forums", "Media"

2. **Approved Tags**
    - **Created by:** Admins (from existing normal tags)
    - **Purpose:** Verified, important tags with higher authority
    - **Appearance:** {<AdminIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />} icon
    - **Quality:** Admin-verified for accuracy and relevance

3. **Standardized Tags**
    - **Created by:** Admins
    - **Purpose:** Core themes and essential categories
    - **Appearance:** {<ApprovedTagIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />}  icon
    - **Consistency:** Ensures uniform categorization across the platform

4. **AI-Recommended Tags**
    - **Created by:** AI model
    - **Purpose:** These are automatically generated, users can add these to the actual tags of a resource
    - **Appearance:** These only exist within the context of the  [right sidebar](/content/guide/03-components/02-right-sidebar)
    - **Consistency:** AI generated, need to be verified by a human user

<Callout title="Tag Hierarchy">
Standardized Tags → Approved Tags → Normal Tags → AI-Recommended Tags
Higher-level tags carry more authority and are considered more important for organizing content.
</Callout>

## Browsing Tags

### Tag List View

### For Regular Users

The tags page displays all available tags with:

- **Tag names** - Listed alphabetically
- **Usage counts** - Numbers on the right showing how many resources use each tag
- **Search functionality** - Find specific tags quickly
- **Pagination** - Navigate through multiple pages of tags

### For Admins

Alongside the normal tag functionality, admins can:

- **Delete Tag** - By clicking the {<DeleteIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />}  icon
- **Edit Tag name** - By clicking the {<EditIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />}  icon
- **Approve Tag** - By clicking the {<ApproveTagIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />}  icon. This converts a normal tag into an approved tag
- **Standardize Tag** By clicking the {<AdminIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />}  icon. This converts an approved tag into a standardized tag
- **Merge Tags** - By clicking the {<MergeIcon className="w-5 h-5 inline align-text-bottom" fill="#777" />}  icon. This merges two tags, the resources related to the second tag are now related to the first tag

<Callout title="Important: Merging Tags">
Please do note that when merging tags, the second tag is merged into the first tag, not the other way around. Thus, the merge button should be interpreted as "Merge Into".
</Callout>

### Using the Search Bar

- Click in the **Search** field at the top
- Type part of a tag name you're looking for
- Results filter automatically as you type
- Clear the search to see all tags again

## Creating New Tags

### For Regular Users

- **Locate the "New tag" field** at the top of the page
- **Type your tag name** - Choose something descriptive and unique
- **Click the plus (+) button** to add the tag
- **Your new tag appears** in the list as a normal tag

### For Admins

- **Locate the "New standardized tag" field** at the top right of the page
- **Type your tag name** - Choose something descriptive and unique
- **Click the plus (+) button** to add the tag
- **Your new tag appears** in the list as a standardized tag


## Tag Creation Best Practices

### Good Tag names:

1. Descriptive and clear ("Citizen Participation")
2. Consistent with existing tags
3. Specific enough to be useful
4. Professional and standardized terminology

### Avoid:

1. Duplicate tags with slight variations
2. Overly broad terms that don't add value
3. Personal or temporary project names
4. Acronyms without context

## Using Tags Effectively

### When Adding Content

1. Start with existing tags - Check if appropriate tags already exist
2. Use multiple relevant tags - Don't limit yourself to just one
3. Consider tag hierarchy - Prefer standardized and approved tags when available
4. Create new tags sparingly - Only when existing tags don't fit

### When Searching Content

1. Browse by tag - Click any tag to see associated resources
2. Combine with other searches - Use tags alongside text searches
3. Follow tag patterns - Notice how similar content is tagged
4. Use high-authority tags - Standardized and approved tags often yield better results

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Tag already exists but slightly different | • Use the existing tag rather than creating a duplicate<br/>• Contact admin about merging if necessary |
| Unsure which tag to use | • Browse similar content to see tagging patterns<br/>• When in doubt, use broader, established tags |
| Tag has no associated content | • This is normal for newly created tags<br/>• Content will accumulate as users apply the tag |
